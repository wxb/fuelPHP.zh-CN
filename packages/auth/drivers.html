Auth 套件

Auth 套件在 Fuel 裡提供一個標準化的認證介面。 這使得我們的使用者能編寫自己的驅動程序，並且輕鬆地將新的驅動程序與舊的程式碼整合， 以保持基本方法一致。

撰寫你自己的驅動

撰寫驅動可以兩種方式達成。透過擴充 Auth 驅動類別以從零開始建立一個新驅動， 或擴充一個存在的類別並在需要的地方變更它們。

一個 Auth 驅動包含三種類型的類別一起運作，這些是：

  • Login（Auth_Login_Driver）。
  • Group（Auth_Group_Driver）。
  • Acl（Auth_Acl_Driver）。

擴充認證類別

像這樣擴充這些類別：

<?php

class Auth_Login_Mydriver extends \Auth\Auth_Login_Driver
{

	// 在驅動類別裡實現所有需要的函式和變數定義。

}

/* End of file app/classes/auth/login/mydriver.php */

實現函式

當擴充 Auth 驅動類別時，注意每一個需要實現的函式的預期回傳值。 每一個方法提供一個範例，如此你知道需要回傳什麼。下面是一個範例： 

/**
 * 取得目前登入使用者的 ID
 * 以此形式：array(driver_id, user_id)
 *
 * @return  array
 */
abstract public function get_user_id();

上面的程式碼告訴你每一個驅動都應該有 get_user_id 函式。 就像所有你擴充的有抽象函式的類別，這些函式必須在擴充的類別中被定義。 此函式的回傳類型提供在註解中，在這裡是 array(driver_id, user_id)。 為了讓你的驅動與其他驅動並肩合作， 確認你回傳正確的值或你的驅動中斷協定。

這裡是相同的函式來自 Simpleauth 驅動的實現。 

/**
 * 取得使用者的 ID
 *
 * @return  Array  包含此驅動 ID ＆使用者 ID
 */
public function get_user_id()
{
	if (empty($this->user))
	{
		return false;
	}

	return array($this->id, (int) $this->user['id']);
}

一旦你讓新驅動的一切都到位，取代或添加到 auth 配置。 

// 在 config/auth.php 中的陣列

// 添加做為第二個驅動
'driver' => array('Simpleauth', 'Mydriver'),

// 或取代存在的驅動
'driver' => array('Mydriver'),
