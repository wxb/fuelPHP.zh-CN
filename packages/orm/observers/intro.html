Orm

Orm 是 物件關聯對映（Object Relational Mapper） 的簡寫，它做兩件事：
對應你資料庫裡的資料列到物件， 並能讓你在這些物件之間建立關係。
它緊隨 活動記錄模式（ Active Record Pattern），但也受到其他系統的影響。

觀察者：簡介

一個事件基底系統能讓你添加行為到指定事件。當一個事件被觀察， Orm 將自動呼叫所有添加的觀察者，去看是否有任何事情要它們去做。

  • 添加觀察者到你的模型
  • 包含的觀察者
  • 建立你自己的觀察者

添加觀察者到你的模型

你可以用兩種方式添加觀察者：只添加名稱讓觀察者在所有事件被呼叫， 或帶著觀察者做為名稱，以及一個要讓觀察者被呼叫的特定事件陣列。
當觀察者與模型在相同命名空間並且以 Observer_ 前綴時， 你可以略去 "Observer_" 前綴，其他情況下你必須提供完整的類別名稱。

class Model_Article
{
	protected static $_observers = array(
		'example', // 將為所有事件呼叫 Observer_Example 類別
		'Orm\\Observer_CreatedOn' => array(
			'events' => array('before_insert'), // 將只在 before_insert 事件呼叫 Orm\Observer_CreatedOn
		)
	);
}
