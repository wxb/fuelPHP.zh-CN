SimpleAuth - 簡介

Simpleauth，正如它的名字所暗示的，是一個被包含在 Auth 套件的簡易認證系統。 除了做為一個運行中的 Auth 實現，也是其他 Auth 驅動程序的範例。 當建立自己的驅動程序時，你可以做為參考使用。

Auth 設置

配置開始於告知 Auth 套件你即將使用 Simpleauth 驅動。 這是透過 auth.php 配置檔案做到。預設檔案在 Auth 套件中被提供。 在做任何變更前，你應該複製此檔案到你的 app/config 文件夾。 預設檔案已經為 Simpleauth 套件配置。你將在此配置檔案的這裡找到說明。

在你完成後，你可以選擇透過 app/config/config.php 的 always_load 段落來自動載入套件。

ACL

SimpleAuth 配備一個支援以下 ACL 結構的 ACL 驅動：

  • 每個使用者是一個群組（對，就是一個）的一員
  • 每個群組可以有零個或多個指派給它的角色
  • 每個角色可以有零個或多個指派給它的權限

當檢查存取時，你指定所需的存取為 role.right 當你想要檢查單一權限， 或 role.[right,right] 當你想要一次檢查多個權限。這是一個 AND 檢查， 所以當你指定多個權限時，該使用者必須要有全部指派權限才能授權存取。

配置

SimpleAuth 認證系統是透過一個配置檔案來配置，不意外地稱為 'simpleauth.php'。 預設檔案在 Auth 套件中有提供。 在你做任何變更前，你應該複製該檔案到你的 app/config 文件夾。

以下配置值可以被定義：

參數 類型 預設 描述
db_connection 字串 
null
 要使用的資料庫連線名稱。這應該與你應用程式 db.php 配置檔案中的定義相符。 設定它為 null 以使用預設 DB 實例。
table_name 字串 
'users'
 要使用的使用者資料表名稱。
table_columns 陣列 
array('*')
 要從使用者資料表選擇的行列表，或 '*' 來選擇所有行。你必須至少包含 'username'、'password'、'email'、'last_login'、'login_hash'、'group' 和 'profile_fields'。
guest_login 布林 
true
 如果為 true，如果沒有登入，假的 'guest' 使用者會被建立。 這能讓你在沒登入時使用群組和 acl 驅動。
remember_me 陣列 
array(
	'enabled' => false,
	'cookie_name' => 'rmcookie',
	'expiration' => 86400*31
)
 給 Simpleauth 'remember_me' 功能的配置
multiple_logins 布林 
false
 如果為 true，多個相同使用者同時登入是允許的。如果為 false，當一個使用者登入，任何先前的登入會被取消。 注意啟用這會停用一些登入 session 劫持的對策！
groups 陣列 
array()
 定義的群組，被 SimpleAuth groups 驅動所使用。 群組陣列結構的敘述，詳見預設配置檔案。
roles 陣列 
array()
 定義的角色，被 SimpleAuth acl 驅動所使用。 角色陣列結構的敘述，詳見預設配置檔案。
login_hash_salt 字串 
'put_some_salt_in_here'
 為了讓 SimpleAuth 驅動使用的密碼額外安全， 一個鹽值被用在儲存密碼進資料庫時雜湊他們。確保你變更預設值為一個非常隨機的字串！ 為了雜湊密碼，SimpleAuth 使用 PBKDF2，一個非常安全的雜湊機制。
username_post_key 字串 
'username'
 在登入表單上包含使用者名稱的輸入欄位名稱。
password_post_key 字串 
'password'
 在登入表單上包含密碼的輸入欄位名稱。

如果你想要使用 'remember-me' 功能，確保你有一個有效的 Crypt 配置， 因為它使用一個加密的 cookie 來儲存要被記憶的使用者資訊。

資料庫表

SimpleAuth 依賴一張資料表，Auth 套件包含要建立此資料表所需的遷移檔案。 只要運行 oil refine migrate --packages=auth 來為你建立該資料表。

範例

這是一個簡單的登入動作：

public function action_login()
{
	$data = array();

	// 如果你按下提交按鈕，讓我們跑整個步驟。
	if (Input::post())
	{
		// 檢查認證，這裡假設你已經建立上表，且
		// 你已經使用如上所述的資料表定義和配置。
		if (Auth::login())
		{
			// 認證成功，進入。
			Response::redirect('success_page');
		}
		else
		{
			// 哎呀，沒你的湯，再試著登入一次。設定一些值來
			// 重填使用者名稱欄位，並給一些錯誤的文字到檢視。
			$data['username']    = Input::post('username');
			$data['login_error'] = 'Wrong username/password combo. Try again';
		}
	}

	// 顯示登入表單。
	echo View::forge('auth/login',$data);
}
