Auth 套件

Auth 套件在 Fuel 裡提供一個標準化的認證介面。 這使得我們的使用者能編寫自己的驅動程序，並且輕鬆地將新的驅動程序與舊的程式碼整合， 以保持基本方法一致。

Auth_Group_Driver

此驅動是所有 Auth 群組驅動的基礎類別。它被定義為一個抽象類別， 該類別包含了通用於所有群組驅動的所有方法， 以及定義給任何群組驅動必須實現的所有抽象方法。

驅動配置

一個群組驅動可以載入它所依賴的額外驅動。一般情況下，這些是 ACL 驅動。 但你的實現情況可以是引進和使用自訂的驅動類型。要做到這一點，添加此結構到你的驅動類別： 

// 當載入此群組驅動時自動載入 Simpleacl acl 驅動
	protected $config = array(
		'drivers' => array('acl' => array('Simpleacl'))
	);

靜態介面

為了易於使用，member() 方法透過通用 Auth 靜態介面曝光。

使用靜態介面，你能做到

// 回傳 true 如果目前使用者是群組 16 的成員。
$is_member = Auth::member(16);

通用方法

通用方法被定義在群組基礎驅動，而且能透過擴充到達所有的 Auth 群組驅動。這些方法提供功能來與群組互動，並檢查存取。

定義在類別中而沒記錄在此的方法是內部使用，而且不應該直接被呼叫。

get_id()

回傳驅動的唯一 ID。這可被用於識別驅動，或用來選擇一個特定的驅動實例。

靜態 否
參數 無
回傳 驅動 ID 字串。
範例 
// 回傳 'simpleauth'
$id = Auth::group()->get_id();

set_config($key, $value)

設定一個驅動配置值。

靜態 否
參數
參數 預設 描述
$key 必要 配置鍵名
$value 必要 此配置鍵的值
回傳 空
範例 
// 在 simplegroup 驅動設定一個配置值
Auth::group()->set_config('key', 'value');

get_config($key, $default = null)

取得一個驅動配置值。

靜態 否
參數
參數 預設 描述
$key 必要 配置鍵名
$default null 如果請求的鍵不存在時要回傳的預設值
回傳 混合
範例 
// 取得一個 simplegroup 驅動的配置值，如果不存在回傳 false
$key = Auth::instance(simplegroup)->get_config('key', false);

抽象方法

你所開發的每個群組驅動必須提供這些所有的方法，且必須回傳此處記錄的值。

member($group, $specific = null)

檢查是否使用者是所給群組中的成員，無論是所有載入的群組驅動， 或只是指定為第二參數的那些。

靜態 是
參數
參數 預設 描述
$group 必要 群組 ID
$specific null null 以檢查所有，或一個或更多驅動 ID（單一字串或字串陣列）
回傳 布林，是否（所給）的驅動之一將目前使用者註冊為一個成員
範例 
if ( ! Auth::member(100) )
{
	Response::redirect('no_admin');
}

// 指定驅動
Auth::member(1, 'simplegroup');

// 多個驅動
Auth::member(0, array('simplegroup', 'genericgroup'));

get_name($group)

取得所給群組名稱的顯示名稱。

靜態 是
參數
參數 預設 描述
$group null 你想要取得顯示名稱的群組名稱
回傳 混合，取回的群組顯示名稱，或 false 如果所給群組不存在。
範例 
// 取得 admin 群組的顯示名稱
Auth::instance('simplegroup')->get_name('admin');
