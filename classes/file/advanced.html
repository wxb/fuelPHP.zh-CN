進階用法

簡介

除了基本的 File 類別的靜態用法，也有較為複雜的選項可用。 首先是區域，基本上它是不能在外部動作的基礎路徑， 並有一些配置選項。當在區域使用 File 類別的方法， 它們將回傳包裝目錄＆檔案的物件。

區域

簡介

檔案區域被用來影響檔案系統行為。透過設定基礎限制， 你可以設定允許在這些限制內靈活運用的界限， 而不須擔心檔案及目錄的未認證的存取。

配置

預設情況下，檔案區域穿過所有目錄無限制存取。有兩種方式來設定這些配置值。 最快的方式是傳遞一個陣列進入 File::forge() 方法。 

$config = array(
	'basedir'		=> DOCROOT.'uploads',
	'use_locks'		=> true,
);

$area = \File::forge($config);

一個更集中的方式是透過在你的 File 配置檔案中定義一個（或更多）區域。要做到這一點， 複製 core/config/file.php 到 app/config/file.php（或建立新的） 並且在 'areas' 陣列中定義你的區域。現在你可以透過提供識別符字串到 File::forge() 方法來取得區域。 

// 在 app/config/file.php 裡面
'areas' => array(
	'my_area' => array(
		'basedir'		=> DOCROOT.'uploads',
		'use_locks'		=> true,
	),
),

// 現在取得區域

$area = \File::instance('my_area');

用法

有兩種方式在 File 類別中使用區域。
首先是在一個靜態情境下，提供一個 File_Area 實例或識別符到你正在呼叫的函式。 

// 讀取目錄的內容
try
{
	$dir = File::read_dir('somedir', 0, null, 'my_area');
}
catch(\FileAccessException $e)
{
	// 操作失敗
}

第二是直接從區域本身呼叫函式。

// 取得區域
$area = File::instance('my_area');

try
{
	$dir = $area->read_dir('somedir', 0, null);
}
catch(\FileAccessException $e)
{
	// 操作失敗
}
