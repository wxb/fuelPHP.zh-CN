Validation 類別

類別幫助你驗證使用者輸入，如果你想要同時建立一個表單＆它的驗證， 使用 Fieldset 類別代替。

類別方法

forge($fieldset = 'default')

forge 方法回傳一個新的 Validation 實例。 並用一個 $fieldset 與 Fieldset 連結。

靜態 是
參數
參數 類型 預設 描述
$fieldset 字串｜Fieldset 
'default'
 名稱或要連結的 Fieldset 實例。
回傳 Validation 物件
拋出 \DomainException，當 Fieldset 名稱或實例已經有一個關聯的 Validation 實例。
範例 
// 建立一個新的 Validation 實例
$val = Validation::forge();

// 關聯新的 Validation 實例與一個Fieldset，使用 Fieldset 名稱
$val = Validation::forge('my_fieldset');

// 關聯新的 Validation 實例與一個Fieldset，透過傳遞 Fieldset 實例
$fieldset = Fieldset::instance('my_fieldset');
$val = Validation::forge($fieldset);

instance($name = null)

instance 方法回傳與 Vaildation 實例關聯的 Fieldset 實例的識別符 $name， 或預設的 Fieldset（如果需要的話會建立）。

靜態 是
參數
參數 類型 預設 描述
$name 字串 
null
 你想回傳的 Validation 的 Fieldset 名稱。
回傳 Validation 物件
或 false 如果指定 Fieldset 實例不存在
範例 
// 取得預設 Fieldset 的 Validation 實例
$val = Validation::instance();

// 取得一個特定 Fieldset 的 Validation 實例
$val = Validation::instance('my_fieldset');

active()

active 方法回傳目前活躍的 Validation 實例。

靜態 是
參數 無
回傳 Validation - 目前活躍的 Validation 實例
範例 
// 取得目前活躍的 Validation 實例
$val = Validation::active();

active_field()

active_field 方法回傳目前被驗證的欄位。

靜態 是
參數 無
回傳 Fieldset_Field - 目前被驗證的欄位
範例 
// 取得目前被驗證的欄位

fieldset()

fieldset 方法回傳關聯的 Fieldset。

靜態 否
參數 無
回傳 Fieldset - 關聯的 Fieldset
範例 
// 取得關聯的 Fieldset
$val = Validation::forge('my_fieldset');

add_field($name, $label, $rules)

add_field 方法是 add() 方法一個更簡單的別名，它可以在一個步驟指定欄位名稱、標籤以及規則。

靜態 否
參數
參數 類型 預設 描述
$name 字串 
無
 要指派驗證規則的欄位名稱。
$label 字串 
無
 欄位的標籤。
$rules 混合 
無
 帶有群組在方括號內的規則參數（選擇性）的驗證規則組， 並用逗號分隔。在被用垂直線或管道符號（|）的規則可被傳遞做為一個字串， 或做為一個規則陣列。
回傳 Fieldset_Field
範例 
// 在不同類型的欄位設定規則
$val->add_field('username', 'Username', 'required|trim|valid_string[alpha,lowercase,numeric]');
$val->add_field('email', 'Email', 'required|trim|valid_email');
$val->add_field('age', 'Age', 'valid_string[numeric]');

// 做為一個陣列
$val->add_field('username', 'Username', array('required', 'trim', 'valid_string[alpha,lowercase,numeric]'));

set_message($rule, $message)

set_message 方法為此 Validation 實例覆寫語言檔案訊息。當沒有訊息被指派到你正在使用的規則時，它也是有用的。

靜態 否
參數
參數 類型 預設 描述
$rule 字串 
無
 要指派此訊息的規則名稱。
$message 字串｜null 
無
 要為此規則顯示的訊息，如果傳遞一個字串，或 null 來回復到語言檔案。
回傳 Validation 實例
範例 
// 覆寫 required 規則訊息
$val->set_message('required', 'You have to fill in your :label so you can proceed');
// ……完成時，設回預設訊息
$val->set_message('required', null);
// 給你建立自訂規則的訊息
$val->set_message('custom_rule', 'The :label you entered is previously registered. Please choose another one');

get_message($rule)

get_message 方法為此 Validation 實例取回一個指定的錯誤訊息。只有透過 set_message 方法設定的訊息會被回傳。

靜態 否
參數
參數 類型 預設 描述
$rule 字串 
無
 你需要取得訊息的規則名稱。
回傳 字串
範例 
// 覆寫需要的規則訊息
$val->set_message('custom_rule', 'The :label you entered is previously registered. Please choose another one');
// ……之後，或在你程式碼中的其他地方，取得此訊息
$val->get_message('required', null);

add_callable($class)

add_callable 方法添加一組自訂或擴充的驗證驗證規則。你不須要寫一個完整的回呼（callback），只要一個類別名稱的字串。這也允許從此物件覆寫功能，因為新類別是附加在前的。

靜態 否
參數
參數 類型 預設 描述
$class 字串｜物件 
無
 做為字串的類別名稱，或物件名稱
回傳 Validation 實例
範例 

$val->add_callable('myvalidation');

為了完整使用此方法的範例，檢查擴充 Validation 類別，以及所需要的類別。

remove_callable($class)

remove_callable 方法從可呼叫陣列移除一個可呼叫。

靜態 否
參數
參數 類型 預設 描述
$class 字串｜物件 
無
 做為字串的類別名稱，或物件名稱
回傳 Validation 實例
範例 
$val->remove_callable('myvalidation');

callables()

callables 方法取回你不需要添加一個完整回呼（callback）但只要方法名稱的物件

靜態 否
參數 無
回傳 陣列

run($input = null, $allow_partial = false, $temp_callables = array())

針對傳遞的 input 運行定義的驗證規則，或 Input::post() 如果沒傳東西的話。

靜態 否
參數
參數 類型 預設 描述
$input 陣列 
null
 input 欄位和值的關聯陣列
$allow_partial 布林 
false
 如果為 false，全部有定義驗證規則的欄位必須在 input 出現
$temp_callables 陣列 
array()
 Any additional callables required to run all defined validation rules
回傳 布林，如果驗證過為 true，如果失敗為 false
範例 
$val = Validation::forge('article');
$val->add_field('title', 'Title', 'required');

if ( ! $val->run())
{
	foreach ($val->error() as $field => $error)
	{
		echo $error->get_message();
		// Title 欄位是必要且必須包含一個值。
	}
}

input($key = null, $default = null)

Returns one or all input values, or the default value if the requested key is not present in the input.

靜態 否
參數
參數 類型 預設 描述
$key 字串 
null
 Name of the input field, or null to return all input
$default 混合 
null
 Value to return if the key is not present in the input
回傳 Mixed, the input value or the value specified as $default
範例 
$val = Validation::forge('article');
$val->add_field('title', 'Title', 'required');

if ($val->run())
{
	// if something was posted, echo it
	echo $val->input('title');
}

Input is only available after the validation rules have run. By default, input() will fall back to Input::param() when no rules have run. This behaviour can be controlled through the validation config file.

validated($field = null, $default = false)

Returns one or all validated input values, or the default value if the requested key is not present in the validated input.

靜態 否
參數
參數 類型 預設 描述
$field 字串 
null
 Name of the input field, or null to return all input
$default 混合 
null
 Value to return if the key is not present in the input
回傳 Mixed, the validated input value or the value specified as $default
範例 
$val = Validation::forge('article');
$val->add_field('title', 'Title', 'required');

if ($val->run())
{
	// if something was posted, echo it
	echo $val->validated('title');
}

Where input() returns the raw input value, validated() will return the validated value, which might be different than the input because some validation rules alter the input (for example a rule like strtoupper).

error($field = null, $default = false)

error 方法回傳在驗證期間發生的指定或所有錯誤。

靜態 否
參數
參數 類型 預設 描述
$field 字串 
null
 你想要回傳的 Validation_Error 實例的欄位名稱。如果你沒指定，會回傳所有錯誤物件。
$default 混合 
false
 如果指定欄位的錯誤不存在，回傳這個值。
回傳 Validation_Error 實例
或 陣列 如果你沒指定欄位名稱
範例 
$val = Validation::forge('article');
$val->add_field('title', 'Title', 'required');

if ( ! $val->run(array()))
{
	foreach ($val->error() as $field => $error)
	{
		echo $error->get_message();
		// Title 欄位是 required 的並且必須包含一個值。
	}
}

error_message($field = null, $default = false)

error_message 方法回傳在驗證期間發生的指定或所有錯誤訊息。

靜態 否
參數
參數 類型 預設 描述
$field 字串 
null
 你想要回傳的錯誤訊息的欄位名稱。如果你沒指定，會回傳所有錯誤訊息。
$default 字串 
false
 如果指定欄位的錯誤不存在，回傳這個值。

或 陣列 如果你沒指定欄位名稱

show_errors($options = array())

show_errors 根據定義的樣板回傳呈現在 HTML 中的驗證錯誤列表。

靜態 否
參數
參數 類型 預設 描述
$options 陣列 
array()
 HTML 樣板的選項，以覆寫來自驗證配置檔案的預設值
回傳 字串，呈現的 HTML
範例 
$val = Validation::forge('article');
$val->add_field('title', 'Title', 'required');

if ( ! $val->run(array()))
{
	echo $val->show_errors();
}

以下選項可以被傳遞：'open_list'、'close_list'、'open_error'、'close_error' 和 'no_errors'。 預設情況下會產生一個無序清單，且如果沒找到驗證錯誤會回傳一個空字串。
