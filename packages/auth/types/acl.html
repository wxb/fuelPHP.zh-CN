Auth 套件

Auth 套件在 Fuel 裡提供一個標準化的認證介面。 這使得我們的使用者能編寫自己的驅動程序，並且輕鬆地將新的驅動程序與舊的程式碼整合， 以保持基本方法一致。

Auth_Acl_Driver

此驅動是所有 Auth acl 驅動的基礎類別。它被定義為一個抽象類別， 該類別包含了通用於所有 acl 驅動的所有方法， 以及定義給任何 acl 驅動必須實現的所有抽象方法。

靜態介面

為了易於使用，has_access() 方法透過通用 Auth 靜態介面曝光。

使用靜態介面，你能做到

// 回傳 true 如果目前使用者有所需的權限
$may_access_comments = Auth::has_access('comments[read,write]');

通用方法

通用方法被定義在 acl 基礎驅動，而且能透過擴充到達所有的 Auth acl 驅動。這些方法提供功能來與特定的 acl 互動，並檢查存取。

定義在類別中而沒記錄在此的方法是內部使用，而且不應該直接被呼叫。

get_id()

回傳驅動的唯一 ID。這可被用於識別驅動，或用來選擇一個特定的驅動實例。

靜態 否
參數 無
回傳 驅動 ID 字串。
範例 
// 回傳 'simpleacl'
$id = Auth::acl()->get_id();

set_config($key, $value)

設定一個驅動配置值。

靜態 否
參數
參數 預設 描述
$key 必要 配置鍵名
$value 必要 此配置鍵的值
回傳 空
範例 
// 在 simpleacl 驅動設定一個配置值
Auth::acl()->set_config('key', 'value');

get_config($key, $default = null)

取得一個驅動配置值。

靜態 否
參數
參數 預設 描述
$key 必要 配置鍵名
$default null 如果請求的鍵不存在時要回傳的預設值
回傳 混合
範例 
// 取得一個 simpleacl 驅動的配置值，如果不存在回傳
$key = Auth::instance(simpleacl)->get_config('key', false);

抽象方法

你所開發的每個 acl 驅動必須提供這些所有的方法，且必須回傳此處記錄的值。

has_access($condition, Array $entity)

has_access 方法使用定義的 ACL 驅動根據所給的條件來檢查使用者的存取。

靜態 否
參數
參數 預設 描述
$condition 必要 在該位置做為字串或陣列，代入一個位置和所需權限。 代入一個位置和所需權限。
$entity 必要 array(driver_id, id) 形式的使用者或群組識別。
回傳 布林，true 如果存取被允許，否則為 false。
範例 
// 檢查使用者是否允許評論
if ( ! Auth::acl()->has_access('comments.create'), array('simplegroup', 1))
{
	Response::redirect('no_admin');
}
