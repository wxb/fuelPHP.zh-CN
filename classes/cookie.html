Cookie 類別

Cookie 類別能讓你取得、設定及刪除 cookies。

配置

Cookie 類別透過全域應用程式配置檔案 app/config/config.php 進行配置。它定義一個稱為 'cookie' 的段落，在其中以下設定被定義：

變數 類型 預設 描述
expiration 整數 
0
 在 cookie 到期前的秒數。當你呼叫 set() 方法，而 $expiration 沒有被指定時，這個值將被使用。
path 字串 
'/'
 限制 cookie 可用的路徑。當你呼叫 set() 方法，而 $path 沒有被指定時，這個值將被使用。
domain 字串 
null
 限制 cookie 可用的網域。當你呼叫 set() 方法，而 $domain 沒有被指定時，這個值將被使用。
secure 布林 
false
 設為 true 如果你只想要透過安全連線傳輸 cookie。
httponly 布林 
false
 只允許透過 HTTP 傳輸 cookie，停用 javascript 存取。

如果全域配置中缺少這些值中的一個或多個時，類別將使用定義在這張表中的預設值。

當使用 cookie，請 務必 確保配置在你的 app/config/config.php 及／或你的 php.ini 檔案中的時區與設定在你伺服器上的時區相符。因為 cookie 的到期時間戳記是 GMT，當你時區有不相符時， 計算到期時間將導致嚴重錯誤，從不正確的過期到 cookie 完全不被設定， 因為它們到達瀏覽器時已經逾期。

get($name = null, $default = null)

get 方法能讓你讀取一個 $_COOKIE 變數。如果沒有給名稱，會回傳所有 cookie。

靜態 是
參數
參數 預設 描述
$name 選擇性 在 $_COOKIE 陣列裡的鍵。
$default 
null
 如果陣列項目沒有找到時應該要回傳什麼值？
回傳 混合
範例 
$theme = Cookie::get('theme', 'blue');

set($name, $value, $expiration = null, $path = null, $domain = null, $secure = null, $httponly = null)

set 方法能讓你建立一個 Cookie。

靜態 是
參數
參數 預設 描述
$name 必要 在 $_COOKIE 陣列裡的鍵。
$value 必要 Cookie 的值。
$expiration 
null
 cookie 應該持續的秒數。
$path 
null
 在伺服器上 cookie 可用的路徑。
$domain 
null
 Cookie 可用的網域。
$secure 
null
 設為 true 如果你只想要透過安全連線傳輸 cookie。
$httponly 
null
 只允許透過 HTTP 傳輸 cookie，停用 javascript 存取。
回傳 布林
範例 
Cookie::set('theme', 'blue', 60 * 60 * 24);

對每個沒指定或定義為 null 的參數，將被替換為全域定義配置值。

delete($name, $path = null, $domain = null, $secure = null, $httponly = null)

delete 方法從 $_COOKIE 陣列刪除一個參數。

當一個 cookie 被建立在指定的路徑或網域等，你也必須提供刪除函式。

靜態 是
參數
參數 預設 描述
$name 必要 移除一個 Cookie 項目。
$path 
null
 在伺服器上 cookie 可用的路徑。
$domain 
null
 Cookie 可用的網域。
$secure 
null
 設為 true 如果你只想要透過安全連線傳輸 cookie。
$httponly 
null
 只允許透過 HTTP 傳輸 cookie，停用 javascript 存取。
回傳 空
範例 
// 刪除通常的 Cookie
\Cookie::delete('theme');

// 設定路徑的 Cookie
\Cookie::set('theme', 'fuel', null, '/account/');

// 刪除該 Cookie
\Cookie::delete('theme', '/account/');
