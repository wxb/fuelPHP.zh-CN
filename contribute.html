簡介

Fuel 是一個社群驅動的框架，並且開放來自任何認為他們可以改進框架或添加功能的人們， 來讓框架到達更高水準的貢獻。 有許多形式可以做到這一點，從寫教學、編寫和發布套件， 以促進成為 Fuel 的正式部分（核心、Orm、文檔等……）。

貢獻程式碼

當你遭遇一個錯誤，或看到問題清單中你能或已解決的問題， 而且你想要將它貢獻回框架。根據你所拉取（pull）正在運行的版本分支， 使用 GitHub 發送一個拉取請求（pull request）。 當你請求一個拉取（pull），依循接下來的幾個步驟：

  • 我目前所使用的發行版本是否仍被支援？
    若否，升級到目前發行版本，並查看錯誤是否仍存在。
  • 錯誤是否仍出現在目前發行版本？
    若是，升級到目前開發分支，並查看錯誤是否修正。
  • 若否，在你的本地儲存庫切換到目前開發分支
    $ git checkout -b 1.7/develop origin/1.7/develop
  • 在你本地的儲存庫修正錯誤並送交（commit）
    使用一個描述性的和能理解的送交（commit）訊息，並確認你簽名了。
    $ git commit -m "This is my awesome description."
  • 將其推送（push）到你的 GitHub 帳戶
    $ git push
  • 請求 pull
    在你所在相同的分支上請求拉取（pull）（＝目前開發分支）。 使用「更新送交（commit）範圍」介面以引導你在正確的分支請求。 拉取請求（pull request）只會在目前發行版本的開發分支（1.5/develop）被接受。
  • 等待你出名的時刻
    只要你的請求被合併，你將會成為名人堂中貢獻者名單的一部分。

你可以在 Github 上找到此程序的詳情。

非錯誤貢獻

非錯誤的貢獻是指新特色、優化以及所有其他的改動。所有這些請求，必須在最新開發分支動作。 所以拉取（pull）最新版本的 1.7/develop 分支進行變更，並請求一個拉取（pull）。

對於大型及／或全新的功能、或將需要一些時間來完成的功能，設置一個獨立的分支可能是明智的做法。 透過 email 或 IRC 聯繫核心開發人員來討論該選項。
