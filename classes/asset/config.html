Asset 類別

Asset 類別是一系列協助收集、分組和顯示資產（js、css、img）的方法。

配置

Asset 類別的預設是透過 fuel/core/config/asset.php 配置檔案來配置。 它已經填入了一個預設配置。 你可以透過添加一個同名的配置檔案到你應用程式的 config 目錄來覆寫此配置，並在這裡設定你想要變更的值。 這些將覆寫核心配置，但保留你沒有覆寫的部份。

以下全域配置值可以被定義：

參數 類型 預設 描述
paths 陣列 
array('assets/')
 預設 asset 根路徑。路徑必須包含一個斜線。
css_dir 字串 
'css/'
 預設 css 子文件夾。必須包含一個斜線。
js_dir 字串 
'js/'
 預設 js 子文件夾。必須包含一個斜線。
img_dir 字串 
'img/'
 預設 image 子文件夾。必須包含一個斜線。
folders 陣列 
array(
	'css' => array(),
	'js'  => array(),
	'img' => array(),
),
為每個資產類型指定一個或更多各自的資產文件夾。
url 字串 
\Config::get('base_url')
 指向你資產的 root URL。當載入一個資產，資產路徑以及資產名稱會被附加到此 URL。
add_mtime 布林 
true
 是否附加資產最後修改時間戳記到 URL，這將有助於快取資產，值得推薦。
indent_level 整數 
1
 為了保持產生的 HTML 整齊，Asset 類可以縮排標籤。這設定縮排的級別。
indent_with 字串 
"\t"
 要用來縮排產生的 HTML 的字串。這預設是一個 tab。
auto_render 布林 
true
 這控制當一個資產方法沒帶群組名稱被呼叫時，是否回傳產生的 HTML。 如果為 false，該資產將被添加到預設群組以在之後呈現。
fail_silently 布林 
false
 當設為 true，當資產檔案沒被找到時，例外將不會被拋出。 該資產將會被忽略。
always_resolve 布林 
false
 當設為 true，Asset 類別始終會嘗試在本地解析資產。當設為 false， it will only resolve it the asset URL is relative (i.e. pointing to the local host itself). This is useful if you have a copy of your assets on a separate asset server, or behind an asset cache like Varnish.

Asset 路徑

有一些方法可以定義資產路徑。你可以定義全域的資產搜尋路徑，它定義你資產的 'root'。 當搜尋一個資產時，為已載入資產定義的子文件夾會被附加到這些路徑。 如果你使用多個全域搜尋路徑，它們必須都使用相同的子文件夾結構。 

// 假設：
// - 你的 URL 是 "http://example.org/"
// - 你的 全域搜尋路徑是 "assets/"
// - 你的 img_dir 是 "img/"
// 這將回傳 <img src="http://example.org/assets/img/icons/myicon.png" />
echo Asset::img('icons/myicon.png');

如果你不想要一個標準的子文件夾結構，或你必須在現有環境的工作， 而該環境的資產路徑已被定義， Asset 類別能讓你為每種支援的資產類型定義個別的資產搜尋文件夾。 

// 如果這是你的配置
// 'folders' => array(
//	'css' => array('assets/css'),
//	'js'  => array('assets/js', 'global/js/'),
//	'img' => array('assets/img/', 'assets/icons'),
// ),
// 而你的圖示在 'assets/icons'，
// 這將回傳 <img src="http://example.org/assets/icons/myicon.png" />
echo Asset::img('myicon.png');

搜尋路徑和搜尋文件夾都可以使用 add_path() 方法在執行階段定義。

靜態資產

如果你的（一些）資產沒儲存在應用程式根目錄，而是在 URL 使用不同主機， 你可以使用下面的技術來載入一個資產： 

// 每個資產個別地
echo Asset::img('http://static.example.org/icons/myicon.png');

// 對所有組別，設定你的 Asset URL 到 'http://static.example.org/'
echo Asset::img('icons/myicon.png');
在這個情況下搜尋路徑會被忽略。
