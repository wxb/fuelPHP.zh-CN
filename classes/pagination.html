Pagination 類別

Pagination 類別能讓你簡單地為你顯示的紀錄設立分頁。

如何使用 Pagination 類別的簡單例子，你可以將它放進你控制器內的 action 方法。

如何使用查訊建立器來分頁

$config = array(
	'pagination_url' => 'http://localhost/fuel/welcome/index/',
	'total_items'    => 10,
	'per_page'       => 5,
	'uri_segment'    => 3,
	// 或者，如果你偏好透過查詢字串的分頁
	//'uri_segment'    => 'page',
);

// 建立一個名為 'mypagination' 的實例
$pagination = Pagination::forge('mypagination', $config);

$data['example_data'] = DB::select('id', 'value')
							->from('pagination')
							->limit($pagination->per_page)
							->offset($pagination->offset)
							->execute()
							->as_array();

// 我們傳遞物件，當在檢視中輸出時它會被呈現
$data['pagination'] = $pagination;

// 回傳檢視
return \View::forge('welcome/index', $data);

如何使用 ORM 來分頁

$config = array(
	'pagination_url' => 'http://localhost/fuel/posts/index/',
	'total_items'    => Model_Post::count(),
	'per_page'       => 10,
	'uri_segment'    => 3,
	// 或者，如果你偏好透過查詢字串的分頁
	//'uri_segment'    => 'page',
);

$pagination = Pagination::forge('mypagination', $config);

$data['example_data'] = Model_Post::query()
							->rows_offset($pagination->offset)
							->rows_limit($pagination->per_page)
							->get();

// 我們傳遞物件，當在檢視中輸出時它會被呈現
$data['pagination'] = $pagination;

// 回傳檢視
return \View::forge('posts/index', $data);

配置

你可以透過幾種方法配置分頁實例。你可以傳遞一個帶有配置的陣列當你鍛造（forge）實例， 或者你可以直接在該實例上更新特性。

可定義以下的配置設定：

參數 類型 預設 描述
pagination_url 字串 
null
 你有分頁頁面的 URL，如果為 null，Fuel 嘗試從目前的 URL 檢測。
uri_segment 整數｜字串 
3
 包含頁碼的 URI 分段（如果為整數）。包含頁碼的查詢字串欄位（如果為字串）。
num_links 整數 
5
 要顯示的連結總數。
total_items 整數 
0
 項目總數。通常這是一個 count() 查詢的結果。
per_page 整數 
10
 每頁筆數。
current_page 整數 
null
 如果 URI 不存在頁碼時要載入的頁面。如果沒給，預設是 1。
show_first 布林 
false
 產生一個「到第一頁」連結如果為 true 並且不在第一頁。
show_last 布林 
false
 產生一個「到最後一頁」連結如果為 true 並且不在最後一頁。
link_offset 整數／浮點數 
0.50
 在分頁區塊中活躍連結的偏移，無論是 0 到 1 之間的小數點，或 0 到 100 之間的整數（百分比）。

連結偏移

預設情況下，Pagination 類別會嘗試定位活躍連結（目前頁面） 在分頁區塊中間。

你可以使用配置中的 link_offset 值來控制此行為。這個值可以定義為 0 到 1 之間的浮點數，或 1 到 100 之間的整數（像百分比）。預設情況下，這個值是設為 0.5 （= 50%）。 如果你讓值變小，活躍連結會往左移動，如果你讓值變大， 活躍連結會往右移動。

範例

讓我們假設你有共 20 頁可以顯示。你已經設定 num_links 為 5，總共顯示 5 頁的連結， 而你顯示上一頁和下一頁連結也是如此。你目前選擇的是第 6 頁。

  • 隨著預設 0.5 的設定，會顯示
    «  4  5  6  7  8  »
    ， 保持選擇的第 6 頁恰好地在區塊中間。
  • 當你變更設定為 0，Pagination 會始終試著讓第一個連結為活躍連結。在這個情況下，
    «  6  7  8  9  10  »
    會被顯示，讓選擇的第 6 頁為區塊中的第一個連結。
  • 如果你想變更設定為 1，Pagination 會始終試著讓最後一個連結為活躍連結。在這個情況下，
    «  2  3  4  5  6  »
    會被顯示。

你可以使用範圍內的任何值，要逐步轉移活躍焦點往左或右。很明顯地，你顯示愈多連結， 就可以愈精細切換焦點。

這只會在有足夠頁面可以這樣做的時候適用。如果你只有 5 頁可用，你想要顯示 5 個連結，而第 3 頁是目前頁面，沒有顯示上一頁和下一頁連結，目前頁面會在中間，不管 link_offset 的設定是什麼。根本沒有足夠的頁面可用來移動目前頁面連結往左或右。

樣板化

每個 Pagination 實例使用一個樣板來產生需要用來建立分頁標記的 HTML。 你可以在 config/pagination.php 檔案中儲存你的標準樣板。 在你做任何修改之前，從核心 config 文件夾複製到你的 app config 文件夾。 預設配置檔案配有三個樣板，FuelPHP 預設以及相容 Twitter Bootstrap v2 和 v3 的樣板。

以下樣板條目必須被定義：

wrapper 字串 
<div class="pagination">\n\t{pagination}\n</div>\n
 將包裹產生的分頁的標記。
first 字串 
<span class="first">\n\t{link}\n</span>\n
 將被用來產生第一頁的標記。
first-inactive 字串 無 將被用來產生第一頁的標記，如果此頁是第一頁或唯一。
first-inactive-link 字串 無 將被用來產生第一頁連結的標記，如果此頁是第一頁或唯一。
first-marker 無 
&laquo;&laquo;
 將被用來產生第一頁標記的標記。
first-link 字串 
\t\t<a href="{uri}">{page}</a>\n
 將被用來產生第一頁連結的標記。
previous 字串 
<span class="previous">\n\t{link}\n</span>\n
 將被用來產生上一頁的標記。
previous-marker 字串 
&laquo;
 將被用來產生上一頁標記的標記。
previous-link 字串 
\t\t<a href="{uri}" rel="prev">{page}</a>\n
 將被用來產生上一頁連結的標記。
previous-inactive 字串 
<span class="previous-inactive">\n\t{link}\n</span>\n
 將被用來為無效連結產生上一頁的標記。
previous-inactive-link 字串 
\t\t<a href="{uri}" rel="prev">{page}</a>\n
 將被用來為無效連結產生上一頁連結的標記。
regular 字串 
<span>\n\t{link}\n</span>\n
 將被用來產生其他頁面的標記。
regular-link 字串 
\t\t<a href="{uri}">{page}</a>\n
 將被用來產生其他頁面連結的標記。
active 字串 
<span class="active">\n\t{link}\n</span>\n
 將被用來產生目前頁面的標記。
active-link 字串 
\t\t<a href="{uri}">{page}</a>\n
 將被用來產生目前頁面連結的標記。
next 字串 
<span class="next">\n\t{link}\n</span>\n
 將被用來產生下一頁的標記。
next-marker 字串 
&raquo;
 將被用來產生下一頁標記的標記。
next-link 字串 
\t\t<a href="{uri}" rel="next">{page}</a>\n
 將被用來產生下一頁連結的標記。
next-inactive 字串 
<span class="next-inactive">\n\t{link}\n</span>\n
 將被用來為無效連結產生下一頁的標記。
next-inactive-link 字串 
\t\t<a href="{uri}" rel="next">{page}</a>\n
 將被用來為無效連結產生下一頁連結的標記。
last 字串 
<span class="last">\n\t{link}\n</span>\n
 將被用來產生最後一頁的標記。
last-marker 字串 
&raquo;&raquo;
 將被用來產生最後一頁標記的標記。
last-link 字串 
\t\t<a href="{uri}">{page}</a>\n
 將被用來產生最後一頁連結的標記。
last-inactive 字串 無 將被用來產生最後一頁的標記，如果此頁是最後一頁或唯一。
last-inactive-link 字串 無 將被用來產生最後一頁連結的標記，如果此頁是最後一頁或唯一。

在樣板中，{uri} 將被產生的分頁連結取代（# 如果是非活躍樣板）， 並且 {page} 被頁碼或上一個／下一個標記取代。 如果你想要在這些標記使用圖像，只要修改定義在樣板中相應的連結， 用圖像標記取代 {page}。

當你鍛造 Pagination 實例時所傳遞的配置將與定義在你配置檔案中的預設樣板合併。 這能讓你指傳遞你想要覆寫值。 如果你在配置檔案中的樣板不完整， 上面所提的預設值將被使用。

forge($name = 'default', $config = array())

forge 方法能建立一個新的分頁實例，並且透過傳遞一個陣列來配置。

靜態 是
參數
參數 預設 描述
$name 必要 要被建立的實例名稱。如果沒有給名稱，'default' 實例會被建立。
$config 
array()
 配置陣列。
回傳 Pagination
範例 
// 建立一個新的分頁實例
$pagination = Pagination::forge('mypagination', array(
	'pagination_url' => 'http://docs.fuelphp.com/',
	'uri_segment' => 2,
	'total_items' => 10,
	'per_page' => 20,
));

instance($name = null)

instance 方法能讓你檢索先前鍛造的實例，或回傳預設實例如果沒給名稱的話。

靜態 是
參數
參數 預設 描述
$name 
null
 要被回傳實例的名稱。如果沒給名稱，'default' 實例會被建立。
回傳 混合，一個 Pagination 物件，或 flase 如果請求的實例不存在。
範例 
// 取回先前鍛造的實例
$pagination = Pagination::instance('mypagination');

render($raw = false)

render 方法產生在檢視中顯示分頁連結的標記。

靜態 否
參數
參數 預設 描述
$raw 
false
 如果為 true，會回傳一個原始分頁資料的陣列，而不是呈現分頁標記。
回傳 字串
範例 
// 取回先前鍛造的實例，並且呈現
echo Pagination::instance('mypagination')->render();

分頁物件包含一個 __toString() 方法，當你寫出該物件、或轉為字串時， 會造成 render() 方法被呼叫。

first($marker = null)

first 方法為分頁產生顯示「第一頁」連結的標記。 如果沒給字串用來做為標記，來自樣板的 "first-marker" 值會被使用。

靜態 否
參數
參數 預設 描述
$marker 
null
 要被顯示在連結裡的文字。預設是 'first-marker' 樣板值。
回傳 字串
範例 
// 取回先前鍛造的實例，並且呈現「第一頁」連結
echo Pagination::instance('mypagination')->first();

「第一頁」連結將只在有第一頁，而你不在第一頁時顯示。 請注意，預設情況下，沒有為 'first' 定義無效連結。

previous($marker = null)

previous 方法為分頁產生顯示「上一頁」連結的標記。 如果沒給字串用來做為標記，來自樣板的 "previous-marker" 值會被使用。

靜態 否
參數
參數 預設 描述
$marker 
null
 要被顯示在連結裡的文字。預設是 'previous-marker' 樣板值。
回傳 字串
範例 
// 取回先前鍛造的實例，並且呈現「上一頁」連結
echo Pagination::instance('mypagination')->previous();

next($marker = null)

next 方法為分頁產生顯示「下一頁」連結的標記。 如果沒給字串用來做為標記，來自樣板的 "next-marker" 值會被使用。

靜態 否
參數
參數 預設 描述
$marker 
null
 要被顯示在連結裡的文字。預設是 'next-marker' 樣板值。
回傳 字串
範例 
// 取回先前鍛造的實例，並且呈現「下一頁」連結
echo Pagination::instance('mypagination')->next();

last($marker = null)

last 方法為分頁產生顯示「最後一頁」連結的標記。 如果沒給字串用來做為標記，來自樣板的 "last-marker" 值會被使用。

靜態 否
參數
參數 預設 描述
$marker 
null
 要被顯示在連結裡的文字。預設是 'last-marker' 樣板值。
回傳 字串
範例 
// 取回先前鍛造的實例，並且呈現「最後一頁」連結
echo Pagination::instance('mypagination')->last();

「最後一頁」連結將只在有任何頁面，而你不在最後一頁時顯示。 請注意，預設情況下，沒有為 'last' 定義無效連結。

pages_render()

pages_render 方法為分頁產生顯示在上一頁及下一頁之間的頁面連結的標記。

靜態 否
參數 無
回傳 混合
範例 
// 取回先前鍛造的實例，並且呈現「pages」連結
echo Pagination::instance('mypagination')->pages_render();

靜態介面

為了方便你， Pagination 類別也有一個只對預設實例操作的靜態介面。

get($name)

get 方法能讓你在預設實例取得一個配置項目。

靜態 是
參數
參數 預設 描述
$name 必要 要取回的特性名稱。
回傳 混合，特性值、或 null 如果特性不存在。
範例 
// 取回目前頁碼
$page_number = Pagination::get('current_page');

// 這是一個別名
$page_number = Pagination::instance()->current_page;

set($name, $value)

set 方法能讓你在預設實例設定一個配置項目為所給的值。

靜態 是
參數
參數 預設 描述
$name 必要 要設定的特性名稱。
$value 必要 要設定的特性值。
回傳 空
範例 
// 設定列總數
$page_number = Pagination::set('total_items', 610);

// 這是一個別名
Pagination::instance()->total_items = 610;

為了向下相容，靜態方法 set_config()、create_links()、 next_link() 和 prev_link() 將被模擬在預設實例， 以最小化升級一個已存在的應用程式到 FuelPHP v1.4+ 的遷移影響。

請注意，現下沒有可能的方法模擬直接存取靜態類別特性， 所以如果你的應用程式使用的話，你將必須改變它們：

// v1.4 前用法：
Pagination::$per_page = 10;

// 新用法：
Pagination::set('per_page', 10);

// v1.4 前用法：
Model_Article::find()
	->order_by('date', 'ASC')
	->rows_offset(\Pagination::$offset)
	->rows_limit(\Pagination::$per_page)
	->get();

// 新用法：
Model_Article::query()
	->order_by('date', 'ASC')
	->rows_offset(\Pagination::get('offset'))
	->rows_limit(\Pagination::get('per_page'))
	->get();
