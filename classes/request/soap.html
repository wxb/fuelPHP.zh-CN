Request_Soap 類別

Request_Curl 類別主要是用來透過 PHP 的 SoapClient 擴充執行 SOAP 請求。 你的 PHP 安裝必須以 "--enable-soap" 編譯。你可以檢查 phpinfo() 的輸出來了解是否是此情況。

建立一個實例

你可以透過 Request 類別鍛造一個此類別的實例： 

// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 請注意，這只建立物件，不會執行請求！

set_params($params)

set_params 方法能讓你在 SOAP 請求呼叫時設定傳遞參數。

靜態 否
參數
參數 預設 描述
$params 必要 給請求的參數陣列。
回傳 Request_Soap，鍊結用
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 設定一些參數
$soap->set_params(array('userid' => 12, 'data' => $payload));

請注意，你必須定義參數的格式取決於被呼叫的 SOAP 服務。 例如，某些情況下期望它們被包裹在一個陣列中。 SOAP Request 驅動將以你在 set_params 使用的格式傳遞那些參數到該服務，所以確保那是正確的。

set_option($option, $value)

set_option 方法能讓你定義一個 SoapClient 選項以被傳遞到該請求。

靜態 否
參數
參數 預設 描述
$option 必要 要設定的 SoapClient 選項。
$value 必要 要設給此選項的值。
回傳 Request_Soap，鍊結用
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 設定用於該請求的使用者名稱
$soap->set_option('login', $username);

set_options(array $options)

set_options 方法能讓你定義多個 SoapClient 選項以被傳遞到該請求。

靜態 否
參數
參數 預設 描述
$options 必要 要設定的 SoapClient 選項和值的陣列。
回傳 Request_Soap，鍊結用
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 設定用於該請求的使用者名稱和密碼
$soap->set_options(array(
	'login' => $username,
	'password' => $password,
	)
);

add_param($param, $value = null)

add_param 方法能讓你添加一個或更多參數到已定義的。

靜態 否
參數
參數 預設 描述
$param 必要 要設定的參數名稱，或一個參數和值的陣列。
$value 
null
 要被定義的值。只用於當 $param 是一個字串值。
回傳 Request_Soap，鍊結用
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 添加一些參數
$soap->add_param(array('userid' => 12, 'data' => $payload));

// 或添加單一參數
$soap->add_param('data', $payload);

set_header($header, $content = null)

set_header 方法能讓你設定一個 HTTP 請求表頭做為該請求的一部分。

靜態 否
參數
參數 預設 描述
$header 必要 HTTP 表頭條目的名稱。
$content 
null
 要被設定的表頭值。
回傳 Request_Soap，鍊結用
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 傳遞一個認證符記到後端伺服器
$soap->set_header('auth-token', 'WV4YaeV8QeWVVVOE');

如果你需要設定一個非 "Name: Value" 形式的表頭，在 $header 傳遞值，並且不要傳遞任何內容。

get_headers()

get_headers 方法能讓你取回所有目前已定義的 HTTP 請求表頭。

靜態 否
參數 無。
回傳 陣列，所有設定的表頭。
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 傳遞一個認證符記到後端伺服器
$soap->set_header('auth-token', 'WV4YaeV8QeWVVVOE');

// 回傳 array('auth-token:WV4YaeV8QeWVVVOE')
$headers = $soap->get_headers();

set_auto_format($auto_format)

set_auto_format 方法能讓你切換自動格式化開關。 Since 1.7.2, this is switched of by default, and when switched off, you will have to parse the cURL response yourself.

靜態 否
參數
參數 預設 描述
$auto_format 必要 True 如果你希望回傳的回應被轉換為一個陣列，false 如果你想要存取它所接收到的。
回傳 Request_Soap，鍊結用
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 我們希望回傳的是一個陣列
$soap->set_auto_format(true);

Auto formatting has support for the following mime types:

  • application/xml
  • text/xml
  • application/json
  • text/json
  • text/csv
  • application/csv
  • application/vnd.php.serialized
so when enabled and the response is in one of these types, it will be processed automatically, and returned to your controller in the form of a PHP array.

Only enable this is the source of the data is trustworthy, and/or if you have validated the received input. JSON and serialized arrays could contain objects. Since their constructor will execute upon instantiation during auto formatting, it may lead to unintended code execution, possibly compromizing your server!

execute(array $additional_params = array())

execute 方法執行定義的 SOAP 請求。

靜態 否
參數
參數 預設 描述
$additional_params 
array()
 任何你想要傳遞給該請求的附加參數。
回傳 Request_Soap，鍊結用
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 執行該請求
$result = $soap->execute();

set_response($body, $status, $mime = null, $headers = array())

set_response 方法設定來自請求中接收的回應。

靜態 否
參數
參數 預設 描述
$body 必要 要被設定做為回應主體的資料。
$status 必要 建立回應的 HTTP 狀態。
$mime 
null
 該資料的 mime 類型。這是用在 auto_format 以轉換該主體為一個陣列。
$headers 
null
 任何你想設定的 HTTP 回應表頭。
回傳 Response，建立的回應物件。
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 建立一個自訂的回應（SOAP 回應始終應為 XML）
$soap->set_response($body, $this->response_info('http_code', 200), 'xml', array('auth-token' => 'WV4YaeV8QeWVVVOE'));

一般情況下，你不應該使用這個方法。它是在請求執行之後用來預處理請求回應物件。

response()

response 方法回傳目前請求的回應。

靜態 否
參數 無。
回傳 Response，建立的回應物件。
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 假設一些參數和選項已經被設定，並且執行
$soap->execute();

// 取回 Response 物件的結果
$result = $soap->response();

response_info($key = null, $default = null)

response_info 方法能讓你取回一個 SOAP 回應值，或所有回應值。

靜態 否
參數
參數 預設 描述
$key 
null
 要取回的特定回應值。當你想要取回所有值時不要指定。
$default 
null
 當該請求值不存在時要被回傳的值。
回傳 混合，取決於請求值的資料類型。
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 假設一些參數和選項已經被設定，並且執行
$soap->execute();

// 從回應表頭取得執行的 soap 動作
$size = $soap->response_info('SOAPAction', false);

此方法允許存取該回應的 HTTP 表頭。

set_function($function)

set_function 方法設定要在 SoapClient 執行的函式。

靜態 否
參數
參數 預設 描述
$function 必要 要呼叫的 SOAP 函式名稱。
回傳 Request_Soap，鍊結用
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 設定 soapcall 函式以取回一些使用者資料
$soap->set_function('GetUserInfo');

get_functions()

get_functions 方法回傳定義在 WSDL 中的函式列表。

靜態 否
參數 無。
回傳 可用的 SOAP 函式陣列
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 取回 SOAP 函式列表
$functions = $soap->get_functions();

這個方法需要 SoapClient 以在 WDSL 模式下運作。

get_request_xml()

get_request_xml 方法回傳上一次請求的 XML。

靜態 否
參數 無。
回傳 字串，上一次的 XML 請求
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 一些程式碼……

// 取回上一次的 XML 請求。
$request = $soap->get_request_xml();

這個方法需要啟用 'trace' 選項。

get_request_headers()

get_request_headers 方法回傳上一次請求的表頭。

靜態 否
參數 無。
回傳 字串，來自上一次請求的 SOAP 表頭。可能包含新行。
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 一些程式碼……

// 取回上一次請求的表頭
$headers = $soap->get_request_headers();

這個方法需要啟用 'trace' 選項。

get_response_headers()

get_response_headers 方法回傳上一次回應的表頭。

靜態 否

回傳 字串，來自上一次回應的 SOAP 表頭。可能包含新行。

get_types()

get_types 方法回傳支援類型的列表。

靜態 否
參數 無。
回傳 SOAP 類型的陣列，詳細描述所有結構和類型。
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 取得對於此 web 服務的 WSDL 中所描述的類型列表
$types = $soap->get_types();

這個方法需要 SoapClient 以在 WDSL 模式下運作。

set_cookie($name, $value = null)

set_cookie 方法為後續請求設定請求 cookie。

靜態 否
參數
參數 預設 描述
$name 必要 cookie 名稱。
$value 
null
 cookie 值。如果沒有指定，該 cookie 將被刪除。
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 在 HTTP 請求表頭中設定一個認證符記做為一個 cookie
$soap->set_cookie('auth-token', 'WV4YaeV8QeWVVVOE');

set_location($location)

set_location 方法能讓你更改端點位置。

靜態 否
參數
參數 預設 描述
$location 必要 新端點的 URL。
範例 
// 建立一個 Request_Soap 物件
$soap = Request::forge('http://soap.example.org/api/v1/this/info', 'soap');

// 更改請求的端點
$soap->set_location('http://moresoap.example.org/api/v2/other/info');
