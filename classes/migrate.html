Migrate 類別

Migrate 類別能讓你從你的控制器運行、穿越與恢復 遷移。 遷移支援應用程式、模組和套件。

current($name = 'default', $type = 'app')

遷移到配置裡的 current 架構設定。

靜態 是
參數
參數 預設 描述
$name 
'default'
 套件或模組的名稱。在應用程式的情況下，使用 'default'。
$type 
'app'
 遷移的類型。有效的值有 'app'、'module' 和 'package'。
回傳 布林
範例 
// 運行應用程式的遷移，到 current 的架構。
Migrate::current('default', 'app');

latest($name = 'default', $type = 'app')

遷移到配置裡的最新架構設定。

靜態 是
參數
參數 預設 描述
$name 
'default'
 套件或模組的名稱。在應用程式的情況下，使用 'default'。
$type 
'app'
 遷移的類型。有效的值有 'app'、'module' 和 'package'。
回傳 布林
範例 
// 運行 'mypackage' 套件的遷移，到最新的架構。
Migrate::latest('mypackage', 'package');

version($version, $name = 'default', $type = 'app')

version 方法將上下移動遷移以設定架構在指定的遷移版本。

靜態 是
參數
參數 預設 描述
$version 必要 你要遷移到的版本，如果你指定 null，它將遷移到最新的版本。
$name 
'default'
 套件或模組的名稱。在應用程式的情況下，使用 'default'。
$type 
'app'
 遷移的類型。有效的值有 'app'、'module' 和 'package'。
回傳 混合 - 遷移到的版本號，或 false 在請求版本是 current 版本的情況下。
範例 
// 遷移 'mymodule' 模組到版本 10
Migrate::version(10, 'mymodule', 'module');
