類別

類別是什麼？

類別就是普通的 PHP 類別。它不需要擴充任何東西，或依循除了命名慣例的任何慣例，Fuel 內的所有其他類別也相同。

class Session

將載入 app/classes/session.php。

載入類別

不像一些其他框架，類別不需要手動載入。當你在程式碼（控制器、模型等）中引用它們，它們將會自動載入。

子目錄中的類別

就像 控制器，類別必須是小寫且第一個字母大寫，同時底線將置類別於子目錄。

Class Session_Driver

將載入 app/classes/session/driver.php

類別與命名空間

為了確定要讀哪個 PHP 檔案，FuelPHP 的自動載入器將把命名空間與使用底線的類別名稱視為完全相同。 這意味著在子目錄裡的類別，你可以混搭命名空間與底線來符合你的需求。

例如有一個類別在 app/classes/core/system/messages.php 檔案裡，此類別可以被定義為：

// 全域的命名空間，完全底線的類別名稱
class Core_System_Messages {}

// 結合一個命名空間與底線
namespace Core;
class System_Messages {}

// 或完全的命名空間
namespace Core\System;
class Messages {}

第一種方法是最常用且最容易理解的，如果你結合 Use 聲明使用已定義的類別， 命名空間的版本是特別方便的。

初始化你的類別

類別被載入時，讓 Fuel 的自動載入器實施某些自動的任務是可能的， 有點像類別的實例 __construct() 所做的。 你可以透過添加一個公開的靜態 _init() 方法到類別達成。

class Example {
	public static function _init()
	{
		// 緊接在載入類別後被呼叫
	}
}

如果一個被載入的類別有它自己的 _init() 方法及父層，必須跟著被載入， 父層的初始化方法不會被呼叫，除非你使用 parent::_init();
