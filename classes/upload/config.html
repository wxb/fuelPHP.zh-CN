Upload 類別

Upload 類別可以安全地處理被上傳到應用程式的檔案。 它能讓你用不同的方式過濾上傳、定義目標檔案名稱應該像怎樣、或過濾檔案名稱的大小或長度。

配置

Upload 類別是透過 fuel/core/config/upload.php 配置檔案進行配置。它已經填充了一個預設配置群組。你可以藉由複製此配置檔案到你的應用程式 config 目錄，並根據需求修改該檔案，以覆寫此配置群組並添加新群組。

以下的配置設定可以被定義：

參數 類型 預設 描述
auto_process 布林 
true
 如果為 true，只要你使用 Upload 類別，上傳的檔案將被使用儲存在配置檔案的配置驗證。
max_size 整數 
0
 每個上傳檔案的最大大小（以位元組為單位）。要停用最大尺寸的限制，設定此參數為 0。
ext_whitelist 陣列 
array()
 允許的副檔名陣列。如果為空或沒定義，所有副檔名都被允許。 副檔名需要被定義為只有小寫。如果上傳的檔案是大寫， 它的副檔名將以小寫字母針對白名單比對。
ext_blacklist 陣列 
array()
 不允許的副檔名陣列。如果為空或沒定義，所有副檔名都被允許。 副檔名需要被定義為只有小寫。如果上傳的檔案是大寫， 它的副檔名將以小寫字母針對黑名單比對。
type_whitelist 陣列 
array()
 允許的檔案類型陣列。如果為空或沒定義，所有檔案類型都被允許。 檔案類型是檔案 MIME 類型斜線前的部份。因此如果 MIME 類型是 "text/plain"，添加 "text" 到此陣列來允許此類型的檔案。
type_blacklist 陣列 
array()
 不允許的檔案類型陣列。如果為空或沒定義，所有檔案類型都被允許。
mime_whitelist 陣列 
array()
 允許的檔案 MIME 類型陣列。如果為空或沒定義，所有檔案 MIME 類型都被允許。
mime_blacklist 陣列 
array()
 不允許的檔案 MIME 類型陣列。如果為空或沒定義，所有檔案 MIME 類型都被允許。
prefix 字串 
''
 當你儲存一個上傳的檔案時，檔案名稱將以此字串前綴。
suffix 字串 
''
 當你儲存一個上傳的檔案時，檔案名稱將以此字串後綴。
extension 字串 
''
 當你儲存一個上傳的檔案時，檔案名稱的副檔名將被此字串取代。
path 字串 
''
 要儲存上傳檔案的路徑。
create_path 布林 
true
 如果為 true，定義的路徑將被建立如果不存在的話。 Upload 類別支援目錄遞迴的建立，因此，如果權限設置妥當，任何路徑可以被建立。
path_chmod 整數 
0777
 在路徑被建立後，要被設定的權限。這個值必須被定義為八進位制，開頭有 0。
file_chmod 整數 
0666
 在上傳檔案被儲存到定義的路徑之後，要被設定的權限。這個值必須被定義為八進位制，開頭有 0。
auto_rename 布林 
true
 如果為 true，在檔案已經存在於指定路徑時，檔案名稱將被後綴一個流水號。該流水號將是下一個可用的數字，從 1 開始。
overwrite 布林 
false
 如果為 true，當儲存時檔案已存在，該檔案將被覆寫。此設定將被忽略如果 'auto_rename' 為 true。
randomize 布林 
false
 如果為 true，當儲存時，檔案將被指派為一個 32 字元的名稱（一個 MD5 字串）。其他設定仍會被套用。
normalize 布林 
false
 如果為 true，檔案名稱將被轉換為一個 ASCII 名稱，並且在儲存時，所有空白會被底線取代。
normalize_separator 字串 
'_'
 如果你已經設定 normalize 為 true ，你可以更改空白要變為什麼，而不是預設的底線。
max_length 整數 
0
 檔案名稱的最大長度。這是計算在所有檔案名稱選項被處理之後，要儲存的檔案名稱，要停用最大尺寸的限制，設定此參數為 0。
new_name 字串 無

新的檔案名稱。此設定沒有設定在預設配置，但支援 Upload::process 配置。新名稱將不會覆蓋前綴和後綴。

注意：上傳多個檔案時，確保 auto_rename 設定是設為 true，以避免多個上傳覆寫相同的目標檔案！

change_case 布林 false

此選項能讓你強迫檔案名稱的大小寫。可能的值有 'upper' 和 'lower'。

警告：如果你打算手動呼叫 Upload::process()，確保 auto_process 是設為 false。否則，上傳的檔案會被處理兩次，而以重複的檔案條目為結果。
