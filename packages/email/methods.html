Email 類別方法

forge($setup = null, array $config = array())

forge 回傳一個新的 Email_Driver 實例基於配置或它接收到的輸入。

靜態 是
參數
參數 類型 預設 描述
$setup 混合 
null
 提供 null 做為預設的配置，一個設定組的名稱，或配置的陣列。
$config 混合 
null
 額外配置陣列以動態修改預設配置。
回傳 一個新的 Email_Driver 實例
範例 
// 簡單的樣本：
$email = \Email::forge();

// 從一個群組載入配置
$email = \Email::forge('my_defaults');

// 提供配置
$email = \Email::forge(array(
	'driver' => 'smtp',
));

// 從群組載入設置以動態覆寫
$email = \Email::forge('my_defaults', array(
	'driver' => 'smtp',
));

Email driver 方法

擴充 Email_Driver 的每個驅動程序都有這些方法。 你所需要的就是喚醒與執行它們。

body($body)

body 轉換輸入為字串，並設定訊息主體。

靜態 否
參數
參數 類型 預設 描述
$body 字串 必要 訊息主體。
回傳 $this
範例 
$email->body('This is my message.');

// 或傳遞檢視給它
$email->body(\View::forge('my/view', $data);

alt_body($alt_body)

alt_body 轉換輸入為字串，並設定訊息的替代主體。

靜態 否
參數
參數 類型 預設 描述
$alt_body 字串 必要 訊息主體。
回傳 $this
範例 
$email->alt_body('This is my alternative message.');

// 或傳遞檢視給它
$email->alt_body(\View::forge('my/alt/view', $data);

priority($priority)

priority 方法設定郵件的優先權。

靜態 否
參數
參數 類型 預設 描述
$priority 字串 必要 優先權： 查看所有郵件優先權。
回傳 $this
範例 
$email->priority(\Email::P_HIGHEST);

html_body($html, $generate_alt = null, $auto_attach = null)

html_body 方法設置郵件主體並且選擇性地從其產生替代的主體。如果指定內嵌的圖片將自動被附加。

請注意： 預設情況下自動附加是開啟的（透過配置）。要關閉這個功能，提供 false 或者更改配置設定。

靜態 否
參數
參數 類型 預設 描述
$html 字串 必要 郵件 HTML。
$generate_alt 布林 
null
 是否產生替代訊息主體，當提供空值時回退（fallback）到預設配置。
$auto_attach 布林 
null
 設定為 true 或 false 以附加嵌入式圖片，當提供空值時回退（fallback）到預設配置。
回傳 $this
範例 
$email->html_body(\View::forge('welcome/email', $data));
// 不產生替代訊息主體
$email->html_body(\View::forge('welcome/email', $data), false);

// 產生替代訊息主體，但不自動附加圖片。
$email->html_body(\View::forge('welcome/email', $data), true, false);

to($email, $name = false)

to 方法增加單一或陣列的地址至 to 接收者陣列。

靜態 否
參數
參數 類型 預設 描述
$email 字串｜陣列 必要 接收電子郵件。
$name 字串 
false
 接收名稱，當 $email 是陣列時忽略。
回傳 $this
範例 
// 添加單一地址
$email->to('me@example.com', 'My Name');

// 添加多個地址
$email->to(array(
	'me@example.com',
	// 帶有名稱
	'me@example.com' => 'His/Her name.',
));

attach($file, $inline = false, $cid = null, $mime = null, $name = null)

attach 方法附加一個檔案。

此方法將會在 attach_paths 陣列裡設定（config/email.php）的附件路徑中搜尋檔案。

靜態 否
參數
參數 類型 預設 描述
$file 字串 
false
 引入檔案的路徑。
$inline 布林 
false
 是否附加內嵌檔案。
$cid 字串 
null
 內容標識符。附加內嵌圖像時使用。
$mime 字串 
null
 MIME 類型預設是去查在 core/config/mimes 內的陣列，以此來覆寫它。
$name 字串 
null
 附件名稱覆寫。
回傳 $this
範例 
// 添加一個附件
$email->attach(DOCROOT.'attachments/report.pdf');

// 添加一個內嵌圖像
$email->attach(DOCROOT.'assets/img/mail/header.png', true, 'cid:headerimage');
// 此圖像將被包成 <img src="cid:headerimage" />

$inline 布林

 MIME 類型預設是去查在 core/config/mimes 內的陣列，以此來覆寫它。

send($validate = null)

send 方法發送信件。

靜態 否
參數
參數 類型 預設 描述
$validate 布林 
null
 是否驗證地址，退回到配置設定。
回傳 布林
範例 
try{
	$email->send();
}
catch(\EmailSendingFailedException $e)
{
	// 驅動程序無法發送信件。
}
catch(\EmailValidationFailedException $e)
{
	// 單一或更多驗證失敗的email地址。
}

get_invalid_addresses()

get_invalid_addresses 方法回傳未通過驗證的email地址。

靜態 否
參數 無
回傳 $this
範例 
try{
	$email->send();
}
catch(\EmailSendingFailedException $e)
{
	// 驅動程序無法發送信件。
}
catch(\EmailValidationFailedException $e)
{
	// 單一或更多驗證失敗的email地址。
	$these_failed = $email->get_invalid_addresses();
}

pipelining($pipelining = true)

pipelining 方法可使用流水線（一次發送多個訊息）。 對於不支援的驅動，這個方法是一個空操作（NOOP）。流水線預設是停用的。

靜態 否
參數
參數 類型 預設 描述
$pipelining 布林 
true
 流水線是否應該被啟用。
回傳 $this
範例 
$email->pipelining(true);
