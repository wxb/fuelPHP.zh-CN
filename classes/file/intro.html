File 類別

簡介

File 類別提供一系列的方法與檔案＆目錄協作。 可以透過一些輔助方法完成，或透過更進階的物件導向方法， 其中所有的檔案和目錄被包裹在物件。

配置

File 類別是高度可配置性的，並給你類別提供行為所需要的所有控制。 限制檔案系統存取，與藉由檔案類型設定驅動， 其配置設定各不相同。

預設的 File 配置被放在 core/config/file.php。要變更這些設定， 建立一個帶有你自己設定的配置檔案在 app/config/file.php。

以下的配置值可以被定義：

參數 類型 預設 描述
basedir 字串 
null
 指向基礎目錄的路徑。預設為 null 沒有限制。 當有定義將限制存取到該指定目錄。
extensions 陣列 
null
 允許擴充的陣列，null 為全部。 如果有定義，File 類別將只處理所給的擴充， 並過濾出其餘部份。
url 字串 
null
 檔案的基礎 url，null 為不可用。
use_locks null 
null
 是否使用檔案鎖。設定為 true 如果你希望 File 類別使用它們。
file_drivers 陣列 
array
 每個檔案驅動的陣列。這些能讓你在檔案類型的基礎上使用自訂驅動。
magic_file 字串 
null
 fileinfo() 魔術檔案的替代位置。如果不指定，系統預設會被使用。
